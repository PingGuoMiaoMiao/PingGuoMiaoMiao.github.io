### 图 5-1：动态组件渲染时序图

**图类型**：UML 时序图（Sequence Diagram）

**应放置位置**：第 5 章 详细设计 - 5.2.7 动态组件渲染功能模块设计

**评审关注点**：
- 组件引用解析到渲染的完整流程是否清晰
- 各个处理阶段的时序关系是否正确
- 占位符替换机制是否合理
- 动态加载的实现方式是否可行

**详细绘制说明（文字版）**：

```
标题：动态组件渲染时序图

参与者（从左到右）：
1. Markdown文件（Markdown File）
2. 解析模块（Parser Module）
3. Markdown解析器（marked库）
4. 组件加载器（Component Loader）
5. Astro组件（Astro Component）
6. 渲染引擎（Render Engine）

时序流程：

[1] Markdown文件 → 解析模块：读取文件内容
    标记：读取包含 @Component.astro (1-10) 的Markdown内容

[2] 解析模块：执行正则表达式匹配
    标记：/@(\w+(?:\.\w+)?)\.astro\s*\((\d+)-(\d+)\)/g

[3] 解析模块 → 解析模块：提取组件引用信息
    标记：{componentName: "Component", startLine: 1, endLine: 10}

[4] 解析模块 → Markdown解析器：替换组件引用为占位符
    标记：将 @Component.astro (1-10) 替换为 <!-- COMPONENT_PLACEHOLDER_0 -->

[5] Markdown解析器 → 解析模块：返回处理后的Markdown
    标记：占位符已插入的Markdown文本

[6] Markdown解析器：解析Markdown为HTML
    标记：marked.parse() 生成HTML结构

[7] 解析模块 → 组件加载器：获取组件引用列表
    标记：传递 [{componentName, startLine, endLine, placeholderIndex}]

[8] 组件加载器 → Astro组件：动态导入组件模块
    标记：import(`../../components/${componentName}.astro`)

[9] Astro组件 → 组件加载器：返回组件实例
    标记：组件准备就绪

[10] 组件加载器 → Astro组件：传递props（startLine, endLine）
    标记：组件接收行号参数

[11] Astro组件 → 组件加载器：返回渲染后的HTML
    标记：组件渲染完成

[12] 组件加载器 → 渲染引擎：替换占位符为组件HTML
    标记：将 <!-- COMPONENT_PLACEHOLDER_0 --> 替换为实际组件HTML

[13] 渲染引擎 → 渲染引擎：合并所有内容
    标记：生成最终的完整HTML页面

[14] 渲染引擎 → 浏览器：输出最终HTML
    标记：页面渲染完成

激活框说明：
- 解析模块：从步骤2到步骤7保持激活
- Markdown解析器：步骤4-6激活
- 组件加载器：步骤7-12激活
- Astro组件：步骤8-11激活

注释框（在关键步骤旁添加）：
- 步骤4：使用占位符避免破坏Markdown解析
- 步骤8：动态导入实现按需加载
- 步骤12：替换机制保证组件正确定位

返回消息使用虚线箭头，表示返回结果。
```
